google汽车场景:
java设计了 汽车开发约定
    interface Icar()
google汽车实现了这个接口
class GooleCar implements Icar();

需求:
将GoogleCar 接入生态圈平台时, 当汽车启动时做一些增强. 增强汽车启动功能.

二次开发, 无法获取源代码, 无法使用继承, 对已经存在的对象上的功能进行增强.
前提:
    可以获取到被装饰的对象GoogleCar实现的所有接口

实现步骤:
1. 自己定义一个装饰类.
2. 实现被装饰对象实现的接口.
3. 为自定义的装饰类传递一个被装饰类的对象
4. 在方法上进行相应的增强.

弊端: 如果被实现的接口中的方法过多, 装饰类中的方法过多冗余
解决: 动态代理
